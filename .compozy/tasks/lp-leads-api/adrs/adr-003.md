# ADR-003: API Controller in `app/Controllers/Api/` Subdirectory

## Status

Accepted

## Date

2026-05-23

## Context

A new controller is needed to serve API responses. The existing controllers (`LandingPagesController`, `LeadsController`, `AuthController`, etc.) all live in `app/Controllers/` with namespace `App\Controllers` and serve HTML views. The new controller must be clearly separated from web controllers to avoid confusion and to enable independent evolution of the API surface.

## Decision

Place the API controller at `app/Controllers/Api/LpController.php` with namespace `App\Controllers\Api`. This creates a single subdirectory `Api/` under `app/Controllers/` to house current and future API controllers.

## Alternatives Considered

### Alternative 1: Flat placement in `app/Controllers/`

- **Description**: Create `app/Controllers/LpApiController.php` with namespace `App\Controllers`, same level as existing controllers
- **Pros**: No new directory; follows existing file layout exactly
- **Cons**: Mixes API controllers with web controllers; as the API grows, the directory becomes cluttered; naming convention (`LpApiController`) is the only differentiator
- **Why rejected**: Naming conventions alone are a weak organizational boundary; a subdirectory makes the separation unambiguous and mirrors common CI4 community practice

### Alternative 2: Full CI4 module structure

- **Description**: Create `app/Modules/Api/` with its own controller, routes, and config files
- **Pros**: Complete isolation; enables independent module testing
- **Cons**: The project does not use CI4 modules; enabling module discovery requires configuration changes; overkill for two endpoints
- **Why rejected**: Already rejected in ADR-001 for the same reasons

## Consequences

### Positive

- `app/Controllers/Api/` is a clear, discoverable home for all future API controllers
- Namespace `App\Controllers\Api` aligns with CI4's PSR-4 autoloading — no additional configuration needed
- Web controllers remain completely untouched

### Negative

- One new subdirectory is created (minor structural change)

### Risks

- None identified — CI4 resolves subdirectory controllers automatically via PSR-4 autoloading

## Implementation Notes

- File: `app/Controllers/Api/LpController.php`
- Namespace: `App\Controllers\Api`
- Class: `LpController extends \CodeIgniter\Controller` (not `BaseController`, which loads session/form helpers unnecessary for a stateless API)
- Route reference: `'Api\LpController::list'` and `'Api\LpController::leads/$1'`
