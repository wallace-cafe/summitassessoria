# ADR-001: Minimal MVP Approach for Login + Dashboard with Landing Pages & Lead Management

## Status

Accepted

## Date

2026-05-19

## Context

We need to deliver a login-protected dashboard for a single business owner to create landing pages and manage leads. The primary success metric is time-to-publish: how quickly a user can go from login to a live landing page collecting leads. We evaluated three approaches with different scope balances.

## Decision

Adopt the **Minimal MVP** approach (Approach 1): a dark-themed admin dashboard where a single business owner logs in securely, creates simple landing pages (title, content, custom slug), and views captured leads in a sortable/filterable table. Each landing page auto-generates a public URL with a contact form that feeds leads directly into the system.

## Alternatives Considered

### Alternative 1: Richer First Release

- **Description**: Everything in Minimal MVP, plus landing page preview, publish/unpublish toggles, inline lead editing, lead status workflow with notes, and dashboard home with stats.
- **Pros**: More complete experience; better visibility into performance.
- **Cons**: Significantly longer time to first version; many features not needed to validate the core loop.
- **Why rejected**: Delays the primary success metric (time-to-publish) and introduces scope risk before the core user flow is validated.

### Alternative 2: Public-First, Dashboard-Second

- **Description**: Landing pages built and deployed first as public-facing pages. Dashboard is minimal: basic lead table and read-only page list.
- **Pros**: Fastest path to live landing pages collecting real leads.
- **Cons**: Dashboard is too thin for ongoing management; does not serve the admin's need to iterate on pages and leads.
- **Why rejected**: The admin's workflow (create page → manage leads) requires a functional dashboard, not just a public form.

## Consequences

### Positive

- Fastest time to validate the core "create page → collect lead → review lead" loop.
- Keeps engineering scope tight and focused.
- Allows user feedback to drive Phase 2 features (preview, stats, inline editing) rather than guessing.

### Negative

- No preview before publishing a landing page.
- No publish/unpublish toggle — pages are live immediately upon creation.
- No inline lead editing; status changes or corrections require future work.
- Dashboard lacks summary stats (total leads, conversion rates) in Phase 1.

### Risks

- Users may publish incomplete pages since there is no preview. Mitigation: add a prominent "Your page is live at this URL" message after creation.
- Lead table may become unwieldy without filtering beyond status. Mitigation: ensure search and sort are robust in the MVP table.

## Implementation Notes

- Use CodeIgniter 4 Filters for session-based login protection.
- Use CodeIgniter 4 Throttler for login rate limiting.
- Use SQLite with WAL mode enabled at database creation for optimized read/write performance.
- Seed the database with a default admin user.
- Follow the DESIGN.md dark theme specification (Binance-style) for the admin dashboard.
